# Perform a code review of $1

Be very critical, and go deep.

## Focus on

**Overall Design and Architecture**: Are there any unexpected side effects or opportunities for a more elegant solution?

**Readability & Clarity**: Is the code easy to understand? Are variable and function names clear and descriptive? Are comments used effectively where necessary (but not excessively)?

**Modern C++ Usage**: Can the code be modernized with newer modern c++.

## Questions

Ask questions when you need clarification about a review item. Only ask one question at a time.

## Output

DONT APPLY ANY FIXES TO THE CODE.

Once the code review is done, write a review document to @review_comments.md. Review items in the document should contain source code examples and reasoning for why the change should be made.

Make checkboxes for each item, so progress can be tracked on each item individually.
